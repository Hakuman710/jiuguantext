# 酒馆内容创建规则

创建角色卡和世界书时，请严格遵守以下约束。

---

## 角色卡 (CharacterCard)

### 必填字段
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识，英文kebab-case，如 `elara-moonshadow` |
| `name` | string | 角色名，中日英均可 |
| `persona` | string | 性格描述，AI会根据此描述扮演角色 |
| `backstory` | string | 背景故事，前80字会作为摘要注入prompt |
| `speechStyle` | string | 说话风格，AI角色的对话风格指引 |

### 属性 (baseStats)
| 属性 | 范围 | 说明 |
|------|------|------|
| `hp` | 0-100 | 生命值 |
| `mp` | 0-100 | 法力值 |
| `atk` | 任意正数 | 攻击力，越高伤害越大 |
| `def` | 任意正数 | 防御力，减免物理伤害 |
| `agi` | 任意正数 | 敏捷力，影响行动顺序等 |
| `spi` | 任意正数 | 精神力，影响魔法效果 |
| `mres` | 0-100 (%) | 魔抗性，百分比减免魔法伤害，上限100 |
| `stamina` | 任意正数 | 体力 |

- **物理伤害公式**: (攻击力²) / (攻击力 + 敌方防御力) × 技能倍率%
- **魔法伤害公式**: 攻击力 × (1 - 敌方魔抗%) × 技能倍率%
- 所有伤害保留小数点后两位

### 技能 (initialSkills)
- 最多 5 个槽位，null 表示空槽
- 每个技能结构：

```json
{
  "id": "skill-id",
  "name": "技能名",
  "type": "physical | magic | buff",
  "mpCost": 0,
  "multiplier": 0,
  "description": "技能描述"
}
```

| 技能类型 | 说明 |
|----------|------|
| `physical` | 物理攻击，走物理伤害公式 |
| `magic` | 魔法攻击，走魔法伤害公式 |
| `buff` | 增益/治疗类，倍率固定为0，不造成伤害 |

- `multiplier` 为百分比倍率（无上限），最终伤害 = 公式结果 × 倍率%
- `mpCost` 为释放消耗的MP值

### 其他
| 字段 | 说明 |
|------|------|
| `affinityBase` | 基础好感度，玩家角色固定从100开始 |
| `tags` | 标签数组，用于分类和搜索 |
| `avatar` | 头像URL（可选，暂未实现图片支持） |

## 世界书 (WorldBook)

### 必填字段
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识，英文kebab-case |
| `name` | string | 世界名称 |
| `description` | string | 世界整体描述，会作为AI系统prompt的第一段 |

### 地点 (locations)
- 每个世界至少1个地点
- `connectedTo` 数组记录可前往的其他地点ID
- 第一个地点默认为玩家的起始位置

### Lore词条 (loreEntries)
- `keys` 是触发词数组，玩家消息中包含任意触发词时自动注入
- `priority` 越高越靠前显示（建议1-10）
- `content` 是注入的Lore知识

### NPC (世界书内置NPC)
- 这些是**世界背景NPC**，不由角色卡控制
- 与角色卡NPC的区别：
  - 世界书NPC：无独立属性面板，仅作为剧情角色存在
  - 角色卡NPC：有完整的属性/技能面板，可在状态面板切换查看
- 玩家在对话中提到NPC名字时，其描述会自动注入AI上下文

### 时间系统 (timeRules)
- `enabled`: 是否启用
- `dayLength`: 现实分钟数 = 1游戏天
- `startHour`: 游戏起始小时(0-23)

---

## 好感度系统
- 范围：0-100
- 玩家角色初始好感度：100（绝对忠诚）
- 当好感度达到100时：
  - 角色绝对服从玩家
  - 好感度永不降低
  - 角色主动维护玩家利益
- 正常情况下每次对话变化不超过±5

---

## 多人模式规则
- 玩家通过 @NPC名 选择哪些NPC参与当前对话
- 只有被@的NPC才会注入prompt，节省上下文
- 被@的多个NPC可以互相交谈
- 已故角色（deadCharacterIds）绝不能以活人形式出现

---

## 格式注意事项
- 所有JSON文件必须是有效的UTF-8编码
- `createdAt` 字段填0即可，导入时自动设为当前时间
- ID命名用英文小写 + 连字符，避免特殊字符
- 角色卡可导出为.json文件（暂不支持PNG嵌入）
